## レビューサマリー / Review Summary

## 良い点 / Strengths
-   各例外処理の目的がコメントで明確に記述されており、コードの意図を理解するのに役立っています。

## 改善点 / Areas for Improvement

### 重要度: 高 / High Priority
-   **例外の発生方法が不適切**: 例外を `throw new NullPointerException();` のように明示的にスローしており、実際に処理の中で例外が発生する状況を再現していません。これは、課題の要件「例外をthrowするのではなく、例外が発生する処理が記述されていること」に反します。
    -   修正案: 各例外が自然発生するような具体的な処理を記述してください。
        -   NullPointerException: `String s = null; s.length();`
        -   ArrayIndexOutOfBoundsException: `int[] arr = new int[0]; int x = arr[0];`
        -   ClassCastException: `Object obj = "Hello"; Integer i = (Integer) obj;`
-   **プログラムが異常終了する**: `catch` ブロック内で例外を再スロー (`throw e;`) しているため、プログラムが異常終了します。これは「try-catchで正常終了すること」という要件に反します。
    -   修正案: `catch` ブロックから `throw e;` の行を削除し、プログラムが例外を捕捉した後に正常終了するように修正してください。
-   **例外の詳細情報が出力されない**: 例外発生時に `System.out.println("例外発生");` のみが出力されており、スタックトレースなどの詳細情報が提供されていません。これは「printStackTraceで詳細情報が出力されていること」という要件に反します。
    -   修正案: `catch` ブロック内で `e.printStackTrace();` を追加し、例外の詳細が出力されるように修正してください。

### 重要度: 中 / Medium Priority
-   **クラス名がJavaの命名規則に準拠していない**: クラス名 `TSUIKA_ENSYU1` が大文字とアンダースコアで構成されており、Javaの標準的なクラス命名規則（PascalCase）に準拠していません。
    -   修正案: クラス名を `TsuiKaEnsyu1` のように修正してください。
-   **汎用的な例外キャッチ**: 各例外の種類が明確であるにもかかわらず、`catch (Exception e)` で汎用的な例外を捕捉しています。これにより、意図しない例外まで捕捉してしまう可能性があります。
    -   修正案: 各 `try-catch` ブロックで、対象となる具体的な例外型（例: `catch (NullPointerException e)`）を捕捉するように修正してください。

### 重要度: 低 / Low Priority
-   **重複する例外処理**: 同様の `try-catch` ブロックが3回繰り返されています。これは演習の目的上、複数の例外パターンを試す意図があるのかもしれませんが、一般的なコードとしては冗長とみなされることがあります。
    -   修正案: 各例外の発生と処理を独立したメソッドに分割することで、`main` メソッドの可読性を高めることを検討してください。
